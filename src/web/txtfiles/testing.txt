
<h2>Testing for the FRAMES User Interface</h2><p>

<b>Background and Scope</b><p>

	The Framework for Risk Analysis in Multimedia Environmental Systems (FRAMES) is a software platform for selecting and implementing environmental software models for risk assessment and management problems. FRAMES User Interface (FUI) allows users to link, select, and interact with environmental models for environmental and human health analyses. formulations for this program are provided in Whelan et al. 1997.<p>

	This document describes how FUI was tested to ensure met its requirements. document lists specific requirements of FUI, shows how test cases follow relate to these requirements, describes each test case, and describes results of running each test case. test cases address basic functionality listed in requirements (black box testing). <p>

<font color="#000080" face="Verdana" size="2"><b>Test Cases and Baseline Testing Results </b><p>

	The purpose of the FUI is to provide a user-friendly and intuitive framework for modeling environmental constituent releases and the resulting human health effects. For this reason, the FUI has many specific requirements that must be met not only to ensure that it functions properly, but also so that it is easy to use and can assist users in setting up a conceptual site model for the problem being addressed. The following test cases were designed to test whether or not the FUI meets all of its requirements. As indicated by the table below, each test case may test several requirements.
