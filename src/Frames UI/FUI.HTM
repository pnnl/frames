Overview

    This section describes how to set up and run cases using the FRAMES (Framework for Risk Analysis in Multimedia Environmental Systems) software system. These instructions assume that the FRAMES user interface is up on the user’s computer.  FRAMES is an open-architecture, object-oriented system that provides an environmental database.  The software aids the user in constructing a conceptual site model that is real world based.Further, the software allows the user to choose the most appropriate models to solve simulation requirements and presents graphical packages for analyzing results.  FRAMES currently contains sockets for a collection of computer models that simulate elements of a source, fate & transport, exposure, and risk-assessment system.
  • Constituent Source and Release to Environment
  • Overland Flow
  • Vadose Zone
  • Saturated Zone
  • Atmospheric
  • Surface Water
  • Exposure Pathways
  • Receptor Intake
  • Human Health Impacts
  • Sensitivity / Uncertainty
  • Ecological Impacts
Specifications for these file types can be found on the FRAMES web page.

    Status Lights

    Moving Icons

    To move an icon, click and drag the icon to the new location.

    Connecting Icons

    To connect icons, the user must hold down the SHIFT key then LEFT-CLICK and drag the arrow from one Icon to the next.

    Disconnecting Icons

    Repeat the SHIFT, LEFT-CLICK used to connect the icons, tracing the same path.

    Selecting a Model

    To select a specific model from a module type icon, first ensure that all module connections are complete, then RIGHT-CLICK and choose General Info. The model selection screen will appear.

    To aid model selection the description of the model has been provided. It is expected that a user will familiarize themselves with any particular model they may choose. FRAMES makes no claims as to the validity of any connections advertised by the models. If a model is selected and OK button pressed, the status light will turn RED.

    Note: This is the best time to name the icon and set the northing, easting and elevation (geo-referernce) information. Changing general information at any time will cause status light changes. The definition of the geo-reference depends on models under each icon type. In general the following definitions apply:

    Definition applicable to the Exposure Pathways icon:
    The Easting, Northing, and Elevation coordinates are in meters and represent the location where exposures occur and assumes that the user has correlated these coordinates with the location of the upstream module.

    1. Upstream module is an atmospheric model:
      1. When <0, 0, 0> is entered by the user – Exposures will be computed at all locations found in the atmospheric output file.
      2. When non-zero values are entered for coordinates by the user – An exposure is only computed at that non-zero location.
    2. Upstream module is not a atmospheric model – The coordinates are not used. Only used for documentation purposes.
    Definition applicable to the Overland Flow icon, Source icon (for MEPAS 5.0 Secondary Source in Soil and MEPAS 5.0 Secondary Source in Standing Surface Water modules only), and Surface Water icon (for Copy of MEPAS 5.0 Secondary Source in Standing Surface Water modules only):
    The Easting, Northing, and Elevation coordinates are in meters and represent the location where calculations apply.
    1. Upstream module is an atmospheric model – Calculations are associated with the single location corresponding to the coordinates entered by the user.
    2. Upstream module is not a atmospheric model – The coordinates are not used. Only used for documentation purposes.
    Definition applicable to the User Defined icon:
    The Easting, Northing, and Elevation coordinates are in meters and represent the location of the centroid of the user defined source module: The coordinates are not used in the module but may be passed along and may be used as the location where ecological exposures occur (e.g., possibly used by SEEM and/or FISHRAND modules). Only used for documentation purposes in this module.

    Definition applicable to ALL other icons:
    The Easting, Northing, and Elevation coordinates are in meters and represent the location of the centroid of the module, if it applies (databases typically do not have locations associated with them): The coordinates are not used in the module. Only used for documentation purposes in this module.

    Model Selection Screen

    Inputting Data into a Model

    To input data into a model from a module type icon, RIGHT-CLICK and choose UserInput.The model’s specific user interface will appear.If the data is entered completely, the status light will be YELLOW.

    Running a Model

    To run a model from a module type icon, RIGHT-CLICK and choose Run Model.The model will run and return to the FRAMES Interface.If the model was run successfully, the status light will be GREEN.

    Deleting an Icon

    To delete an icon, RIGHT-CLICK and choose Delete.

    Naming an Icon

    Name the Module Icon from the General Information Screen by changing the default label.

    Viewing Results

    View results by adding a VIEWER type module to the screen.Then, from the General Information screen, choose a viewer.

    Sensitivity Uncertainty

    To conduct sensitivity or uncertainty analysis on the module input values, place a Sensitivity module on the workspace.Then connect it to all modules that you are interested in either stochastically running the inputs or would like the outputs to be reported.
